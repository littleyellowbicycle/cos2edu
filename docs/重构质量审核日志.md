### 发现的 Bug 及修复

#### Bug P2-1：QuizForm 中 `ws.submitAssessment` 硬编码空参数

**严重程度**：高
**现象**：`QuizForm.vue` 第 110 行 `ws.submitAssessment(quiz.point_id, '', formattedAnswers, 0)` 传空 `characterId` 和 0 的 `conversationId`
**根因**：QuizForm 作为独立组件无法访问 Chat.vue 的 `characterId` 和 `conversationId`
**修复**：
1. 移除 `useWebSocket` 导入和直接 WS 调用
2. QuizForm 通过 `emit('interact', { action: 'submit', value: {...} })` 发送交互事件
3. Chat.vue 中 `<QuizForm @interact="handleQuizInteract" />` 捕获事件
4. `handleQuizInteract` 调用 `ws.submitAssessment(pointId, characterId, answers, conversationId)` 使用正确的参数

#### Bug P2-2：Chat.vue 中 `assessmentAnswers.value = {}` 悬空引用

**严重程度**：中
**现象**：Chat.vue 第 500 行引用已删除的 `assessmentAnswers` ref
**根因**：移除考核面板 JS 代码时遗漏了 WS 事件处理器中的引用
**修复**：移除 `ws.on('assessment.quiz')` 回调中的 `assessmentAnswers.value = {}`，QuizForm 自身的 `watch(quizData)` 会重置答案

#### Bug P2-3：`<QuizForm />` 缺少事件监听

**严重程度**：中
**现象**：Chat.vue 中 `<QuizForm />` 没有绑定 `@interact` 事件，用户提交测验会静默失败
**修复**：改为 `<QuizForm @interact="handleQuizInteract" />` 并添加 `handleQuizInteract` 函数

### 质量审核流程标准化

在本文件创建后，每完成一个重构步骤，执行以下审核：

1. **构建验证**：`vite build` 通过 + 后端 `python -c "from app.engines..."` 通过
2. **僵尸引用扫描**：`grep` 检查移除的变量/函数/CSS 类是否仍被引用
3. **Props 一致性**：对比 registry `propsSchema` vs Vue组件 `defineProps` vs 后端 `TOOL_DEFINITIONS`
4. **CamelCase 转换**：确认后端 `PROP_RENAMES` 覆盖所有 snake_case 参数
5. **事件流完整性**：确认组件 emit 事件有上层监听（Chat.vue 或 DynamicRenderer）
6. **CSS 变量一致性**：确认新组件使用 `var(--color-*)` 而不是硬编码颜色值
7. **空数据安全**：确认每个组件对 optional props 有 `v-if` 或 `default()` 守卫
8. **代码分割验证**：`vite build` 输出中确认新组件为独立 chunk
